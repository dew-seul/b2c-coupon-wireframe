화면 #1

공비서 앱 미입점 샵 안내

B2C에 입점하지 않은 샵의 원장님이 쿠폰 만들기 메뉴에 들어왔을 때 보이는 안내 화면입니다. 쿠폰은 공비서 앱에서 받고 사용되는 기능이라, 입점이 선행 조건이에요. "지금 못 쓴다"가 아니라 "이렇게 하면 쓸 수 있다"의 행동 가이드 형태로 풀었습니다.

의도

  • 막힌 화면이 아닌 다음 행동을 안내하는 화면으로 설계. 권한 부재만 알리면 사용자가 길을 잃기 쉬우니, 입점 가치 + 진입 경로를 한 번에 제시.
  • 혜택 카드 3개로 입점 결정에 필요한 정보를 시각적으로 그룹화. 긴 설득 텍스트보다 빠른 스캔에 유리.
  • 단일 CTA "공비서 앱에 입점하러 가기" 한 개로 결정 부담 최소화. 클릭 시 온라인 예약 정보 페이지로 이동해서 입점 흐름이 끊기지 않게 합니다.

논의 필요

  • 메뉴 위치 선택: 마케팅 → 쿠폰 만들기 진입 시 안내할지, 온라인 예약 메뉴에서 "쿠폰 만들기는 입점 후 사용 가능"식으로 안내할지 고민. 마케팅 진입 시점 안내는 "쿠폰 만들고 싶다"는 사용자 의도에 가깝지만, 입점 자체는 온라인 예약 영역이라 두 메뉴 사이의 흐름이 갈라지는 게 어색할 수 있어요. 한쪽으로 정리해야 메뉴 구조도 깔끔합니다.
  • 혜택 카드 카피: 임시 문안. 실제 마케팅·운영 톤에 맞춰 정리 필요.

화면 #2

쿠폰 목록

쿠폰 만들기 메뉴에 들어왔을 때 가장 먼저 보이는 화면입니다. 만든 쿠폰을 카드 그리드로 한눈에 모아 보고, 새 쿠폰을 만들고 싶으면 페이지 헤드 우측의 "새 쿠폰 만들기" 버튼으로 만들기 페이지로 이동합니다.

의도

  • 카드 그리드로 시각적 스캔성 강화. 테이블 행보다 카드가 쿠폰별 정보(타입·이름·혜택·사용 수)를 묶어 보여줘서 한 눈에 비교하기 쉬워요.
  • 탭 제거, 만들기는 별도 페이지로 분리. 메인 화면은 쿠폰 목록만 보여주고, 새로 만들기는 페이지 이동으로 처리해서 두 흐름이 섞이지 않게 했어요.
  • 완료·중지 쿠폰은 흐리게 표시. 진행 중 쿠폰을 우선 보고, 끝난 쿠폰은 시각적으로 구분되어 정보 위계가 분명합니다.
  • 카드 클릭 시 쿠폰 상세 페이지로 이동. 카드는 핵심 정보만 요약하고, 자세한 발송 내역·사용 이력은 상세 페이지에서 확인할 수 있도록 분리했어요.

정책

  • 발송 전 쿠폰은 수정 가능. 발송 후 쿠폰은 조회 전용 + 중지만 가능.

논의 필요

  • 정렬·필터: 쿠폰이 많아질 때 상태별 필터, 만든 날 정렬 등이 필요할지 검토.

화면 #3

쿠폰 상세

쿠폰 목록에서 카드를 클릭하면 진입하는 페이지입니다. 쿠폰의 모든 정보와 사용 통계를 한 화면에서 확인하고, 상태에 따라 가능한 액션(중지·복제 등)에 바로 접근할 수 있어요.

의도

  • 핵심 통계 우선 노출. 사용 통계(발행·사용·사용률)를 페이지 위쪽에 큰 숫자로 보여줘서, 쿠폰이 잘 쓰이고 있는지 한눈에 파악할 수 있어요.
  • 쿠폰 정보는 라벨–값 형태로 정리. 만들 때 입력한 내용을 그대로 라벨–값으로 보여줘 빠르게 스캔하면서 점검할 수 있어요.
  • 우측 미리보기 유지. 만들기 화면과 같은 위치에 미리보기를 두어, 쿠폰이 실제로 어디에 어떻게 노출되는지 상세에서도 확인할 수 있어요.
  • 상태별 액션을 헤드 우측에 배치. 발송 전 쿠폰은 [수정] [삭제], 발송 후는 [중지]만 노출(기획서 명시 정책). [수정] 클릭 시 만들기 화면으로 이동해서 입력값이 채워진 상태로 편집할 수 있어요. 시안은 발송 후(진행 중) 케이스로 표시.
  • 단독 복제 액션은 M1 범위 외. 기획서엔 캠페인 단위 "이 캠페인 다시 보내기"만 있어, 쿠폰 단독 복제는 M2 검토 예정.

쿠폰 만들기

원장님이 새 쿠폰을 만들 때 보는 메인 화면입니다. 직접 보내기와 공비서에 걸어두기 두 방식을 한 화면에서 모두 처리할 수 있도록 통합했어요. 발송 방식을 바꾸면 받을 대상·미리보기·CTA 라벨까지 함께 바뀌도록 해서, 모드별 결과를 입력 중에 확인할 수 있도록 설계했습니다.

의도

  • 5단계로 나뉜 입력을 한 화면으로 통합. 단계별로 페이지를 옮기는 방식은 입력값 보존·뒤로 가기·앞 단계 수정이 번거로운데, 한 화면에서 모든 입력을 보면서 동시에 조정할 수 있게 해서 입력 부담을 줄였어요.
  • 즉시 반영 미리보기. 이름·혜택·기간 입력이 곧바로 우측 미리보기에 반영되어 시행착오를 줄이고, 처음 만드는 원장님도 결과를 학습하면서 만들 수 있어요.
  • 걸어두기 default. 새 고객 유치(B2C 앱 노출)가 단골 대상 발송보다 더 큰 가치라는 가정. 직접 보내기는 단골 대상 보조 흐름으로 둡니다.
  • 사용 기간 두 가지 방법 제공. 받은 날부터 며칠 동안(7·14·30일) 쓸 수 있게 할지 정하거나, 달력에서 "5월 31일까지"처럼 특정 종료일을 직접 고를 수 있어요. 받은 시점부터 N일 흐름(단골 재방문)과 정해진 날짜까지(캠페인) 흐름을 모두 커버하기 위해서예요.

논의 필요

  • "완료" 상태 정의: 걸어두기 자동 종료 조건이 모호. 일정 기간이 지나면 자동 종료할지 / 발급된 쿠폰의 사용 기간이 모두 만료되었을 때를 완료로 볼지 트리거 정의가 필요해요.
  • 시술 카테고리 연동: 우리 샵에 등록된 시술을 그대로 가져오되 행사상품·티켓은 빼기로 했어요. 콕예약을 함께 보여줄지는 콕예약이 마케팅 대상이 되는지에 따라 결정 필요합니다.
  • 두 방식 혼합 일관성: 일수와 종료일 두 방식이 같은 "사용 기간"이지만 의미가 다릅니다(사용자별 기간 vs 절대 종료일). 원장님이 차이를 헷갈리지 않도록 헬퍼 텍스트나 보조 안내가 필요할 수 있어요.

쿠폰 만들기 완료 모달

쿠폰을 만든 직후에 뜨는 완료 모달입니다. 만들기 끝이 곧 끝이 아니라 다음 행동(보내기 / 확인)의 시작점이라는 흐름을 강조했어요.

의도

  • 다음 행동 강조. 모달을 닫는 부수 액션(저장만/닫기)보다 다음 흐름(바로 보내기 / 공비서에서 보기)으로 이어주는 메인 버튼을 시각적으로 더 강조.
  • 요약 박스. 방금 만든 쿠폰의 핵심 정보(이름·혜택·적용 범위·사용 기간)를 한눈에 보여줘서, 잘못 만든 경우 즉시 발견할 수 있어요.
  • 발송 방식별 다른 메인 액션. 직접 보내기는 다음 단계(받는 사람 선택)로, 걸어두기는 "공비서에서 보기"로 분기. 걸어두기 시 클릭하면 공비서 샵 상세 페이지를 새 창으로 열어, 모달과 만들기 흐름을 끊지 않고 결과만 확인할 수 있어요.

※ 시안에선 두 변형(직접 보내기·걸어두기)을 나란히 비교 노출. 실제 운영에선 원장님이 선택한 발송 방식에 따라 한 쪽만 보입니다.

단체 메시지 전송 화면에서 "새 템플릿 만들기"를 누르면 노출되는 모달입니다. 기존 카카오톡 템플릿 만들기 흐름에 쿠폰 첨부 기능을 새로 추가했어요. (문자 템플릿도 같은 패턴 적용)

의도

  • 쿠폰 첨부 = ON/OFF 토글로 단순화. 메시지 유형을 따로 고르지 않아도 토글 하나로 쿠폰을 붙이고 떼고 할 수 있어요.
  • 본문 자동 템플릿 잠금. 쿠폰 첨부 ON 시 쿠폰 전용 템플릿이 자동 적용되고 본문 작성은 잠겨요. 카카오 템플릿 검수 통과를 위한 정형 문구 보장. 변수칩은 노출되지 않음 (메시지 작성 불가하기 때문).
  • 미리보기 즉시 반영. 쿠폰 카드 + 변수 치환된 본문이 미리보기에 그대로 표시되어 발송 결과를 입력 중에 확인 가능.

논의 필요

  • 쿠폰 + 이미지 동시 첨부 가능 여부: 카카오톡 알림톡 정책상 이미지와 쿠폰 카드를 함께 띄울 수 있는지 확인 필요. 불가능하면 쿠폰 첨부 ON 시 이미지 영역을 자동 비활성화해야 합니다.
  • 링크 버튼 처리: 쿠폰 첨부 시 기존 "예약 링크 버튼"을 그대로 둘지, "쿠폰 다운로드 버튼"으로 변경해 B2C 쿠폰 다운로드 페이지로 보낼지 결정 필요. 후자가 사용자 흐름(쿠폰 받기 → 사용)에 더 자연스러워요.
  • 커스텀 링크 가능 여부: 쿠폰 첨부 시 커스텀형 링크 옵션을 비활성화해 흐름을 표준화할지, 자유 설정으로 두고 원장님 판단에 맡길지 결정 필요.

추가한 내용

  • 메시지 정보 카드에 "첨부 쿠폰" 행 추가. 쿠폰 뱃지 + 쿠폰 이름을 한 줄로 보여줘서, 어떤 쿠폰이 첨부됐는지 즉시 확인할 수 있어요.
  • 성과 요약에 "쿠폰 사용 / 발급" KPI 추가. 사용 수 + 발급 수 + 사용률을 같이 보여줘서, 쿠폰 효과를 메시지 성과와 함께 비교 가능.
  • 예약 고객 테이블 "결제 금액"에 "쿠폰 사용" 뱃지 추가. 쿠폰을 사용해 결제한 행에만 금액 옆에 작은 뱃지를 붙여서, 어떤 결제가 쿠폰 사용 결과인지 한 눈에 보여요.

의도

  • 쿠폰 사용 여부는 별도 컬럼 대신 결제 금액 옆 뱃지로. 쿠폰은 '결제 시 적용되는 혜택'이라 결제 금액과 의미적으로 결합돼 있어요. 컬럼 늘리지 않으면서 시선 흐름(금액 → 쿠폰 적용 여부)이 한 셀에서 처리되도록.
  • 결제 금액에 할인 금액은 별도 표기하지 않음. 결제 금액은 이미 할인이 반영된 실 매출이라 SSOT로 다루고, 쿠폰 종류(정액/정률/증정)별로 표시 형태가 달라지는 복잡도도 피했어요. 할인 효과 집계는 위 KPI에서 사용 수·사용률로 확인.

화면 #9

예약 상세 — 원장님 직접 발급 쿠폰

원장님이 발행한 쿠폰(직접 보내기 / 공비서에 걸어두기)으로 들어온 예약의 상세 화면이에요. 예약금 아래에 "쿠폰 정보" 섹션을 두어, 어떤 쿠폰으로 예약됐는지 확인하고 캠페인까지 드릴다운할 수 있게 했어요. PC·APP 동일 패턴.

쿠폰 정보 섹션

  • 위치: 예약금 바로 아래.
  • 구성: 쿠폰명 + 할인 금액을 한 줄로 노출. 우측 "자세히 보기" → 캠페인 상세로 드릴다운.

왜 이렇게 만들었나

  • info bar·강조 색 X: 본인이 발행한 쿠폰이라 알림성 강조는 과잉. 정보성 한 줄 row로 충분.
  • 네이버 쿠폰과 동일 UI 패턴: "쿠폰 정보" 섹션 명칭·row 구조를 통일해 학습 비용 줄이고 시선 흐름 단순화.
  • "자세히 보기"로 분리: 한 줄에는 쿠폰명·할인 금액만, 발송일·유효기간·대상 같은 메타는 캠페인 상세로 깊이 분리해 화면 비대화 방지.

예약금 유형별 결제 처리

  • 예약금 전액 (선결제): 모든 시술 금액을 예약 시 미리 결제. 쿠폰 할인이 PG 결제에 즉시 반영 → 할인된 금액으로 결제. 결제 완료 = 쿠폰 사용 확정.
  • 예약금 일부 (고정 금액 · 가장 일반): 예약금 설정 샵의 99.2%가 고정, 그 중 92%가 1~2만원. 쿠폰 할인은 총 시술 금액에서 차감, 예약금은 변동 없이 그대로 결제. 현장 결제 = 할인 후 시술 금액 − 예약금. 엣지: 예약금이 할인 후 금액 이상이면 예약금을 할인 후 금액으로 자동 조정 + 안내.
  • 예약금 없이 예약: 쿠폰 적용 상태로 예약 생성. 매출 등록 시 자동 적용 → 매출 등록 완료 시 쿠폰 확정.

예외 처리

  • 예약 취소: 예약금 환불 + 쿠폰 복원 (유효기간 내).
  • 노쇼: 쿠폰 사용 처리 (복원 X).
  • 최소 예약 금액 미달: 쿠폰 적용 불가 안내.

매출 등록 · 공비서 쿠폰 자동 인식

시나리오는 공비서 자체 예약 + 공비서 B2C 쿠폰 사용 케이스. 네이버 예약 정보 카드는 노출되지 않아요. (네이버 예약 케이스에서만 그 카드가 노출됨)

추가한 내용

  • "최종 결제" 카드에 "쿠폰" 행 자동 표시. 고객이 발급받아 사용한 공비서 쿠폰을 매출 등록 시 자동 인식해서, 어떤 쿠폰이 어느 시술에 얼마만큼 적용됐는지 보여줘요. ("쿠폰 사용" 뱃지 + 쿠폰명 · 적용 시술 · 할인 금액)
  • 쿠폰 적용 시술도 "수정/할인" 모달 진입은 가능. 시술 가격이나 직접 할인은 자유롭게 편집할 수 있어요. 단, 모달 안의 쿠폰 자동 적용 부분(쿠폰 할인 금액)만 잠금 처리 (#11 참고).

의도

  • 쿠폰을 "최종 결제" 안에 노출한 이유. 쿠폰은 결제 시 적용되는 자동 할인이라 결제 요약과 의미적으로 결합돼 있어요. 시술/제품/쿠폰을 같은 영역에서 한 줄씩 보면 원장님이 "이 매출에 어떤 항목들이 어떻게 더해지고 빠졌는지" 한눈에 파악 가능.
  • 금액 수정은 열어두되, 쿠폰 자동 적용 값만 잠금. 기획서 §B(line 287) "할인 쿠폰의 자동 적용 값은 원장님이 임의 수정/삭제 불가" 룰을 모달 안 쿠폰 행 단위로만 적용. 시술 가격이나 추가 직접 할인은 정상 편집 가능.
  • 매출 저장 시점이 곧 쿠폰 사용 확정. 별도 확정 단계 없이 매출 등록과 함께 처리돼요 (기획서 line 245).

화면 #11

금액 수정/할인 모달

매출 등록 화면에서 시술별 "수정/할인" 버튼 클릭 시 뜨는 모달이에요. 단일 시술의 금액·할인을 편집해요.

구성

  • 시술 정보 헤더: 어떤 시술을 편집 중인지 + 수량 표시. (이 케이스는 쿠폰 적용 시술 = 손 > 젤네일 기본2)
  • 총 금액 입력: 1개당 단가 표시 + 직접 금액 수정. 편집 가능.
  • 할인 입력: 퍼센트(%) / 정액(원) 라디오 + 입력값. 편집 가능.
  • 쿠폰 행 (NEW): 자동 적용된 쿠폰명 + "자동 적용 · 수정 불가" 안내 + 할인 금액(read-only, 회색 배경). 잠금.
  • 최종 금액 카드: 총 금액 − 직접 할인 − 쿠폰 할인 = 결과 (gray-700 강조).
  • "할인 조건 다른 시술 일괄 적용" 체크박스: 같은 직접 할인을 다른 시술에도 적용. (쿠폰 행은 일괄 적용 대상 아님)
  • 메모: 기록용 텍스트 영역.

쿠폰과의 관계

  • 금액 수정은 자유롭게, 쿠폰 부분만 잠금. 시술 가격(총 금액)과 직접 할인(퍼센트/정액)은 원장님이 정상 편집 가능. 쿠폰 자동 적용 값만 read-only로 표시.
  • 잠금 근거: 기획서 §B(line 287) "할인 쿠폰의 자동 적용 값은 임의 수정/삭제 불가". 진입 단계가 아니라 모달 안 쿠폰 행 단위에서 제어.
  • 쿠폰 미적용 시술에서는 쿠폰 행 자체가 노출되지 않음. 일반 케이스(총 금액 + 할인 + 최종 금액 + 일괄 적용 + 메모)로만 작동.
